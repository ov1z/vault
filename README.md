## Vault — ローカル暗号化パスワードマネージャー

Pr完全ローカル・オフラインで動くパスワードマネージャー。
データはこの PC 内の 1 ファイルにaes-256でで暗号化保存されます。

- UI: HTML/CSS/JS を Windows 標準の **Edge WebView2** ネイティブウィンドウで表示（pywebview）
- バックエンド: Python（`cryptography`）
- 配布: **PyInstaller で単一 .exe**

> ⚠️ **免責**: これは個人プロジェクトで、第三者によるセキュリティ監査は受けていません。
> 標準的な暗号プリミティブを使っていますが、利用は自己責任でお願いいたします。
> 本ソフトウェアは無保証で提供されます。

## セキュリティ設計

| 項目 | 内容 |
|------|------|
| 鍵導出 | マスターパスワード (+ PIM) → **PBKDF2-HMAC-SHA512** → 256bit 鍵（旧 scrypt 保管庫も読込可） |
| 暗号化 | **AES-256-GCM**（改ざん検知付き）。保管庫全体を 1 つの暗号文として保存 |
| パスワード検証 | verifier トークンで誤パスワード/PIM を即検出（全件復号せず判定） |
| メタデータ保護 | タイトル・URL・メモも含め平文では保存されない |
| 書き込み | 一時ファイル → `os.replace` の原子的書き込みで破損を防止 |
| マスター鍵 | アンロック中のみメモリ上に保持。ロック / 終了で破棄 |

### PIM（Personal Iterations Multiplier） — VeraCrypt 方式
PBKDF2 の反復回数を PIM で制御する第2の秘密。**PIM 自体はファイルに保存されない**ため、
パスワードと PIM の両方が正しくないと復号できない（間違えてもどちらが違うかは表示しない）。

```
PIM 空欄(0)  →  反復回数 500,000（標準）
PIM > 0      →  反復回数 = 15,000 + PIM × 1,000
```

VeraCrypt 非システムボリュームと同じ式です。**注意**: この式では PIM が 485 未満だと
標準(500,000)より反復回数が少なく＝弱くなります（VeraCrypt と同じ挙動）。アプリの設定画面では
実際の反復回数と「標準より強い/弱い」をその場で表示します。より強くしたいなら PIM を大きく
（例: 600 → 615,000 回）。

> ⚠️ **マスターパスワード（と設定した PIM）は復元できません。** 忘れると保管庫は二度と開けません。
> `store.json` のバックアップを取っておくことを推奨します（暗号化済みなのでそのままコピーで可）。
>
> 既存の scrypt 保管庫はそのまま開けます。PIM を使いたい場合は「マスターパスワード変更」で
> 新形式（PBKDF2+PIM）へ移行できます。

### 漏洩対策（UI 層の防御）
- **URL スキーム制限**: `http(s)` のみをクリック可能リンクにし、`javascript:` 等は不活性テキストとして表示
  （特権コンテキストでの任意コード実行を防止）
- **Content-Security-Policy**: `script-src 'self'` / `connect-src 'none'`。インラインスクリプト実行・
  外部への通信を不許可（万一の XSS でも外部送信できない）
- **クリップボード**: コピーしたパスワードを Windows の**クリップボード履歴(Win+V)・クラウド同期から除外**し、
  30 秒後に自動クリア（その値のままの場合のみ）
- バックエンドに**ネットワーク送信コードは一切なし**（漏洩の出口がない）

### 守れる範囲 / 守れない範囲（脅威モデル）
- ✅ 守れる: `store.json` の盗難・紛失（暗号化済み・マスターパスワード必須）、改ざん（GCM 検知）、
  クリップボード履歴/クラウド経由の残留、URL 経由のコード実行
- ❌ 守れない: **すでに侵害された OS**（マルウェア/キーロガー/メモリダンプ/管理者権限の攻撃者）。
  アンロック中はメモリ上に平文の鍵・項目が存在し、Python ではメモリの完全消去ができないため。
  これはローカル型パスワードマネージャー共通の制約です。

## データの保存場所

```
%USERPROFILE%\.vault\store.json
```

環境変数 `VAULT_HOME` でディレクトリを変更できます（例: USB メモリに置く）。

```powershell
$env:VAULT_HOME = "E:\my-vault"
```

## 開発環境で動かす

```powershell
cd c:\Users\Nasu\x\vault
python -m pip install -r requirements.txt
python run.py
```

`VAULT_DEBUG=1` で WebView の開発者ツールが開きます。

### バックエンドのテスト
```powershell
python test_backend.py     # 暗号・CRUD・改ざん検知（GUIなし）
python smoke_gui.py        # 実ウィンドウを数秒開いてブリッジ往復を検証
```

---

## .exe をビルドする

```powershell
cd c:\Users\Nasu\x\vault
.\build.ps1
# 出力: dist\Vault.exe
```

> ビルドは `python -m PyInstaller` 経由で実行されます（依存が入っている Python 3.12 を確実に使うため）。
> `pyinstaller` コマンドを直接叩くと、別バージョンの Python が選ばれて依存が欠けることがあります。

できた `dist\Vault.exe` は単体で配布・実行できます。

### 配布時の注意
- **WebView2 ランタイム**が必要です（Windows 11 には標準で同梱、Win10 でも Edge 導入済みなら大抵入っています）。
- 署名していない exe なので、初回起動時に **SmartScreen の警告**が出ます（「詳細情報」→「実行」）。

## 機能
- マスターパスワードでロック / アンロック
- ログイン項目の追加・編集・削除（タイトル / ユーザー名 / パスワード / URL / メモ）
- インクリメンタル検索
- パスワード表示トグル & ワンクリックコピー
- 強力なパスワード生成（長さ・文字種・紛らわしい文字除外）
- **2FA / TOTP コード生成**（base32 シークレットまたは `otpauth://` URI を登録すると、
  ワンタイムコードを 30 秒ごとに自動更新＆残り秒数リング表示。RFC 6238 準拠・stdlib のみ）
- **自動ロック**（無操作 1/5/15/30 分でロック。メニューで設定、`Ctrl+L` で即ロック）
- **PIM 対応**（VeraCrypt 方式。パスワードに加えた第2の秘密で KDF 反復回数を制御）
- マスターパスワード / PIM の変更（保管庫を再暗号化）

### UI
硬派モノクロ・ダーク。アクセントは控えめな青 1 色、アイコンはすべて線画 SVG（絵文字不使用）。

## 構成
```
vault/
├─ run.py                 # 起動エントリ（PyInstaller のターゲット）
├─ build.ps1              # exe ビルドスクリプト
├─ requirements.txt
├─ test_backend.py        # ヘッドレス・バックエンドテスト
├─ smoke_gui.py           # GUI スモークテスト
└─ vaultpm/
   ├─ crypto.py           # scrypt + AES-256-GCM
   ├─ store.py            # 暗号化ストレージ + CRUD
   ├─ pwgen.py            # パスワード生成
   ├─ totp.py             # TOTP (RFC 6238, 2FA コード生成)
   ├─ clipboard.py        # クリップボード（Win32 / posix）
   ├─ api.py              # JS から呼ぶ API（window.pywebview.api）
   ├─ app.py              # WebView ウィンドウ生成
   └─ web/                # フロントエンド（HTML/CSS/JS）
```
